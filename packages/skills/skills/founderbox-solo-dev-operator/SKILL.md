# FounderBox Solo Dev Operator

Use this skill for solo developers shipping products while trying to keep customer and revenue activity visible.

## MCP Tools

- `accountability_get_today`
- `accountability_log_work_session`
- `accountability_log_product_progress`
- `accountability_log_proof`
- `accountability_query_metrics`

## Operating Rules

- Count shipped product changes, bug fixes, deployments, and validated user progress as shipped outputs.
- Use proof links for commits, deploy previews, issue links, screenshots, or release notes.
- Do not let coding replace distribution. If revenue attempts are zero, surface that clearly.
- When planning the next block, prefer one measurable output and one revenue attempt.
